## Project background

**Project**: Road to Summer / 盛夏之路 - Hermes Fitness Agent Extension  
**Repository**: https://github.com/Starryyu77/aicoach-codex  
**Current commit**: `9b5feb1 Add Hermes fitness extension scaffold`  
**Domain**: A Hermes-based fitness training Agent extension with a Road to Summer skill pack, UI Gateway, Training Cockpit frontend shell, ASR mock, Vision/Pose mock, and structured output contracts.

This project is deliberately not a standalone fitness app, coach workstation, SaaS backend, CRM, scheduling system, payment system, or membership system. Hermes is expected to own Agent Runtime, Memory, Sessions, Skills, Tools, model calls, and long-term learning. Road to Summer adds the fitness domain layer: Skill Pack, input/output adapters, training cockpit UI, voice/video interfaces, and training-card presentation.

### 2. UI Gateway

Location:

```text
road-to-summer/gateway/
```

The Gateway is not a traditional backend and does not use a database. It is an adapter layer between the frontend and Hermes.

Responsibilities:

- Receive frontend text input.
- Receive voice input and call ASR.
- Receive camera/video input and call Vision/Pose Tool.
- Build Hermes messages with current session context.
- Call Hermes through `HermesClient`.
- Parse and validate Hermes JSON.
- Map Hermes output into frontend UI state.
- Cache lightweight local state only.

Gateway API:

```text
POST /chat
POST /voice/transcribe
POST /vision/assess
GET  /session/current
POST /session/start
POST /session/end
GET  /history
GET  /history/:id
GET  /memory
POST /memory/confirm
```

Allowed lightweight files:

```text
current_session.json
current_plan.json
training_cards/*.json
mock_memory.json
```

No PostgreSQL, Supabase, MongoDB, or custom memory runtime is used.

## How to run

From repository root:

```bash
npm test
npm run gateway
```

Gateway starts at:

```text
http://127.0.0.1:8787
```

Example calls:

```bash
curl -X POST http://127.0.0.1:8787/chat \
  -H 'content-type: application/json' \
  -d '{"text":"今天该练什么？"}'

curl -X POST http://127.0.0.1:8787/vision/assess \
  -H 'content-type: application/json' \
  -d '{"exercise":"高位下拉","media":"mock-frame","provider":"mock"}'
```

Frontend:

```bash
cd road-to-summer/frontend
npm install
npm run dev
```

Note: frontend dependencies were not installed in the current local run. The frontend shell is implemented, but the verified test coverage is Gateway-level and Node-level.
