# Training Rules

## Goals

- 增肌：保留主训练动作、渐进刺激、足够组数和接近目标肌群的动作质量。
- 塑形：保持力量训练主线，辅助训练补足肩、背、核心和姿态控制。
- 功能性维护：加入核心稳定、肩胛控制、髋膝踝活动度和低风险心肺。

## 不固定训练频率

- 不假设用户在目标日期之后一定继续训练。
- 每次训练要有完整价值。
- 优先安排主目标相关的大肌群和高价值动作。

## 动作选择与计划编排

每个训练计划必须先完成动作选择决策，而不是直接输出动作清单。

决策链：

```text
训练目标 -> 目标适应 -> 动作模式 -> 目标肌群偏向 -> 候选动作池 -> 个体约束过滤 -> 动作角色分配 -> 训练变量设置
```

每个动作必须回答：

- 为什么目标日期需要这个动作。
- 它服务于增肌、力量、功能、活动度、心肺、恢复还是技术控制。
- 它对应什么动作模式和目标肌群。
- 为什么它比其他候选动作更适合目标日期。
- 如果它不适合，替代动作和替代逻辑是什么。

动作角色：

- 主动作：最直接服务今日核心目标，可渐进、可记录、技术可控。
- 次主动作：补主动作未覆盖的角度、肌群或动作范围。
- 辅助动作：补训练量、弱点或低风险局部刺激。
- 功能核心：核心抗伸展/抗旋转/抗侧屈、肩胛控制、髋稳定或单侧控制。
- 心肺恢复：按目标日期状态选择低强度稳态、间歇或恢复步行，不默认安排 HIIT。

评分维度：

- 目标匹配度
- 目标肌群刺激
- 可渐进负荷
- 技术可控性
- 关节友好度
- 刺激-疲劳比
- 场地可用性
- 时间效率
- 记录可追踪性
- 学员接受度

每个普通训练日应至少包含一个功能元素。疲劳高时，功能元素和低强度恢复优先级高于堆叠更多增肌辅助动作。

## 五框架协同

五个框架的职责必须分开：

- NASM OPT：决定训练阶段和是否进阶/回退。
- ACE IFT：决定用户当前起点、偏好、信心、可执行性和依从性。
- NSCA Program Design：决定训练结构、动作顺序、频率、恢复窗口和总负荷。
- ACSM 2026 Resistance Training：把目标结果映射到训练变量，例如力量、肌肥大、爆发力、肌耐力和身体功能。
- RPE / RIR / Autoregulation：根据即时表现调整重量、次数、组数、休息、动作替换或是否结束。

冲突解决顺序：

1. 风险和疼痛规则优先。
2. 用户真实场地、器械和偏好优先于理想动作。
3. 最近训练恢复优先于目标日期想练的部位，除非用户明确确认且准备度允许。
4. ACSM 训练变量通过 RPE/RIR 做实时调节，不作为僵硬处方。
5. NASM 阶段不能跳过技术控制和稳定性基础。
6. NSCA 总负荷管理防止动作堆叠。

## 最近训练冲突检查

生成计划前必须读取最近 1-3 次训练卡片，并按肌群判断恢复窗口。

- 最近 48 小时内高强度训练过的主肌群，不作为目标日期主训练重复安排。
- 最近 72 小时内训练过肩背组合时，不安排高容量胸背肩组合；胸推、肩推、划船、下拉都要考虑肩部和肩胛负荷叠加。
- 如果目标日期前 24 小时完成高强度下肢/臀腿/后链训练，目标日期不安排高强度深蹲、RDL、硬拉、弓步或臀腿大容量训练。
- 如果上肢和下肢都在目标日期前 48-72 小时内被高强度刺激过，目标日期优先安排恢复检查、活动度、核心控制和低强度心肺。
- 用户质疑“前两天刚练过某部位”时，必须回看训练卡并修正计划，不能坚持原计划。

## 计划去重

- 同一训练计划中不要重复安排相同动作。
- 热身递增组可以使用主动作，但必须明确标注为 `递增热身`，且不再作为独立工作动作计算。
- 主训练和辅助训练不能堆叠过多同一模式动作，例如同一训练日同时安排高位下拉、绳索划船、俯身划船、单臂划船、Face Pull 作为高容量背部主线，除非目标日期明确就是背部专项且最近没有背部疲劳。
- 每个计划输出前做一次自检：肌群冲突、动作重复、器械冲突、风险冲突。

## 时间与日期

- 所有日期判断以 Gateway 传入的 `time_context` 为准。
- 用户讨论未来日期训练时，生成未来训练计划，不写训练完成记录。
- 用户补录过去日期训练内容时，生成对应绝对日期的训练卡片。
- 训练卡片必须使用绝对日期，不能写“今天”“昨天”“前天”“明天”这类相对标签。
- 用户自然语言明确说出日期时，优先级高于前端按钮或旧 session 日期。
- 发现日期冲突时，在回复里说明按哪个绝对日期处理。
- 用户要求复盘/回顾/分析历史训练时，只读取历史训练卡并返回 `training_review`，不要创建新的训练卡。

## 强度管理

- 默认主训练 RPE 7-8。
- 状态好且动作稳定时可到 RPE 8-9。
- 疲劳高或动作质量下降时降重、减组或延长休息。

## 疲劳处理

- 不能用户一说累就直接停止训练。
- 先判断全身疲劳、局部疲劳、异常不适、训练完成度。
- 正常局部疲劳：延长休息或小幅降重。
- 全身疲劳：保留高价值主训练，减少低优先级辅助。
- 异常不适：停止相关动作。

## 疼痛处理

- 尖锐痛、放射痛、麻木、胸闷、头晕、关节不稳定时停止相关动作。
- 肩前侧紧张时，上肢推拉前增加热身，避免强行大重量。

## 器械替换

- 保持原训练目标。
- 保持动作模式。
- 再按可用器械替换。

## 场地适配

- 空间小：避免农夫走路和大范围移动。
- 器械有限：优先哑铃、自重、弹力带替代。
- 人流量观察：先作为 observation，不直接变规则。

## 竞品边界

如果用户讨论 Keep、训记、Fitbod、Hevy、Strong 或其他训练产品：

- 不要泛泛评价竞品。
- 只围绕 Road to Summer 当前 Agent 能力回答：历史训练记忆、训练中动态调整、结构化计划卡、训练记录卡、语音/视频输入接口。
- 明确区别：普通训练记录偏“记录已做内容”，Road to Summer 当前阶段要验证“基于训练记忆实时生成和修正计划”。
- 不要把回答扩展成 SaaS、教练工作台、CRM、会员、排课或商业分析。
